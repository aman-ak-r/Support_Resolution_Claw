# Support Resolution Claw 🦅 — v2

An autonomous AI agent pipeline built with **LangGraph**, **FAISS**, and **Sentence-Transformers** to assist micro-entrepreneurs and merchant partners. The system semantically searches a local FAQ/SOP knowledge base, either resolves the query via a self-verified answer, or escalates it into a persistent structured ticket using hybrid scoring, circuit breaker retries, and Pydantic schema contracts.

---

## What's New in v2

- **Pydantic Contracts** — Every node validates inputs and outputs against explicit schemas in `agent/schemas.py`.
- **Self-Verification Step** — A structurally separate LLM call verifies the answer is grounded in the retrieved context before it is returned. Failed verification routes to escalation.
- **Loop/Retry Circuit Breaker** — Up to 2 attempts per query. On the 2nd failure, the pipeline forces escalation.
- **Persistent Ticket Lifecycle** — Escalations generate UUID-keyed tickets with state `open → assigned → resolved → closed` stored in SQLite.
- **SQLite Fallback Queue** — If the database write fails, the interaction is queued in-memory and retried automatically on the next operation.
- **Eko-Specific Knowledge Base** — 26 SOP documents covering CSP registration, transaction timeouts, KYC re-verification, wallet blocks, and settlement delays.
- **Edge-Case Test Suite** — 8 pytest tests covering low-confidence routing, high-risk keyword escalation, retrieval failures, malformed LLM output, and SQLite failure fallbacks.

---

## Key Features
- **LangGraph Routing**: Stateful graph with retrieval → decision → answer → verification → resolution/escalation.
- **Hybrid Confidence Score**: Blends FAISS vector distance and LLM self-rating.
- **Self-Verification Node**: Independent LLM call with no conversation history validates answers before returning them.
- **Persistent Ticket State Machine**: `open → assigned → resolved → closed` with full transition timeline history.
- **Relational Logging with Fallback**: SQLite + in-memory queue if disk write fails.
- **Interactive UI**: Streamlit dashboard with live telemetry, ticket state controls, and database viewer.

## Quick Setup

### 1. Prerequisites
Python **3.9 to 3.11** required.

### 2. Install Dependencies
```bash
pip install -r requirements.txt
```

### 3. Environment Configuration
Default mode is `Mock` — no API keys required. To switch providers, edit `.env`:
```bash
# OpenAI
LLM_PROVIDER=openai
OPENAI_API_KEY=your-api-key

# Gemini
LLM_PROVIDER=gemini
GEMINI_API_KEY=your-api-key
```

## Running the Application

### 1. Build the FAISS Vector Index
```bash
python -c "from agent.retrieval import KnowledgeBaseRetriever; r=KnowledgeBaseRetriever(); r.build_and_save_index()"
```
*(Or click **⚙️ Rebuild Vector Index** in the Streamlit sidebar.)*

### 2. Run the Streamlit Dashboard
```bash
streamlit run app.py
```
Open `http://localhost:8501`.

---

## Running Unit Tests
```bash
pytest tests/
```
Runs 8 tests across two files:
- `test_decision.py` — confidence scoring and routing
- `test_harness.py` — edge cases: retrieval failure, malformed output, SQLite fallback, high-risk keywords

---

## SQLite Database
Auto-created at `logs/support_claw.db` with two tables:
- **`interactions`** — every query, confidence scores, status, and response
- **`tickets`** — persistent escalation tickets with UUID, severity, state, and transition history
